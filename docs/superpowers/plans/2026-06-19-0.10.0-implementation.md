- [ ] **Step 3: Create the hoisting pass**

Create `src/core/hoist.rs`:

The pass walks the core expression tree. When it finds a `Let` binding whose
RHS is a block containing lambda members (the namespace pattern), it:

1. Extracts each lambda member as a new top-level `Let` binding
2. Gives it a mangled name (e.g. the original namespace key)
3. Replaces `Lookup(namespace_var, "member")` references with direct variable
   references to the hoisted binding

Key design decisions:
- Only hoist lambdas, not data members
- Only hoist from known namespace blocks (str, cal, io, etc.)
- Preserve the original namespace block (it may be used as a value)
- The hoisted bindings should be placed at the same scope level as the namespace

Study how `distribute()` in `src/core/inline/reduce.rs` collects inlinables
from `Let` bindings to ensure the hoisted bindings are in the right position
for `distribute()` to find them.

```rust
use std::path::{Path, PathBuf};
use std::process::Command;

/// Cache directory for bare git clones
fn cache_dir() -> PathBuf {
    dirs::home_dir()
        .unwrap_or_else(|| PathBuf::from("."))
        .join(".eu")
        .join("cache")
        .join("git")
}

/// Resolve a git import to a local filesystem path.
///
/// Clones the repository (bare) if not cached, then extracts the
/// requested file at the specified commit.
pub fn resolve_git_import(
    url: &str,
    commit: &str,
    path: &str,
) -> Result<PathBuf, GitImportError> {
    let url_hash = hash_url(url);
    let repo_dir = cache_dir().join(&url_hash);
    let extract_dir = cache_dir().join(format!("{}/{}", url_hash, commit));
    let target = extract_dir.join(path);

    // Return cached file if it exists
    if target.exists() {
        return Ok(target);
    }

    // Clone bare repo if not cached
    if !repo_dir.join("HEAD").exists() {
        clone_bare(url, &repo_dir)?;
    }

    // Extract file at commit
    extract_file(&repo_dir, commit, path, &extract_dir)?;

    if target.exists() {
        Ok(target)
    } else {
        Err(GitImportError::PathNotFound {
            path: path.to_string(),
            commit: commit.to_string(),
        })
    }
}
```

Implement `clone_bare()` using `git clone --bare`, `extract_file()` using
`git archive`, and `hash_url()` using a simple hash of the URL string.

- [ ] **Step 4: Write the analysis pass**

Create `src/core/analyse_demand.rs`:

Backward abstract interpretation over the core expression tree:

```rust
/// Analyse demands on bindings and compute per-function demand signatures.
///
/// Runs after inline/fuse, before STG compile.
pub fn analyse_demands(expr: &mut RcExpr) {
    // Walk the expression tree bottom-up
    // For each binding, compute its demand based on how it's used:
    //
    // - Var reference: demand is (S, AtMostOnce) if single use, (S, Multi) otherwise
    // - Case scrutinee: strict
    // - Application f(x): propagate f's demand signature to x
    // - Let/LetRec: fixed-point iteration for recursive bindings
    // - Lambda body: collect argument demands into DemandSignature
}
```

Key rules:
- `Case(scrutinee, branches)`: scrutinee gets `Strict`; branch bodies are joined
- `App(f, arg)`: if f has a known signature, arg gets the corresponding demand;
  otherwise arg gets `(L, Multi)` (conservative)
- `Let(bindings, body)`: analyse body, then propagate demands to each binding's
  RHS
- `LetRec(bindings, body)`: iterate until fixed point (lattice is finite,
  guaranteed to converge)
- `Lam(args, body)`: analyse body, collect demands on args into `DemandSignature`
